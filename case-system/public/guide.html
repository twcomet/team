系統使用說明

📌 整體流程概覽

每個案件依照以下順序流轉，並有唯一的案件編號（格式：HX年月-序號，例如 HX2605-001）

💬 LINE 詢問 → 📝 詢價階段 → 🔍 場勘階段 → ✅ 成交階段
重要：每個階段的案件分別在不同頁面管理。狀態會決定案件出現在哪個頁面，推進後就會移到下一個頁面。

💬 LINE OA 詢問管理

接收客戶透過 LINE 官方帳號傳來的諮詢訊息，是案件的起點。

狀態怎麼流轉

一通詢問會在這幾個狀態間移動：
新詢問 →（客服點「標記進行中」）→ 進行中 →（點「轉為案件」）→ 已轉案 →（案件走完）→ 結案 / 無效。
重點：每位客人只有一個對話視窗，狀態只是讓它換位置（換分頁），不會因為轉案就多開一個新視窗。

狀態說明
新詢問客戶剛傳訊息、系統自動建立，還沒有人處理。客人最後發言且還沒回 → 會顯示「🔴 待回覆」
進行中客服正在處理。只是換個標籤，還沒有建立任何案件
已轉案已經開了一筆正式案件（有案件編號、進入估價／場勘／報價／派工流程）
結案已轉案，且連結的案件已結案（順利完工結束）
無效客戶無意願／作廢，不繼續跟進
隱藏暫時隱藏，不顯示在列表

💡 已結束的視窗（無效／結案）若客人又傳新訊息，會自動變回「新詢問」，用同一個視窗接續，不會另開。

進行中 vs 已轉案 — 差在哪？

比較進行中已轉案
本質詢問上的一個標籤真的開了一筆案件
觸發「標記進行中」按鈕「轉為案件」按鈕
有案件編號 / 雲端資料夾沒有都會自動建立
進入案件流程還沒已接入（估價／場勘／報價／派工／收款）

🤖 轉真人客服是什麼？

這不是一個狀態，是 AI 打的「提醒標記」。AI 讀完對話後，若判斷「這通不該只由 AI 回、需要真人接手」，就會標記起來、集中到左側「🤖 轉真人客服」分頁，讓客服優先處理（可疊加在「新詢問」或「進行中」的任何一通上）。詳情頁會出現紅色「⚠️ 建議轉真人客服：原因」。

會被標記的情況怎麼處理
問到公司機密（有幾個人、成本、毛利、進貨來源、報價底價、其他客戶資料）一律不回答內容，禮貌帶過並轉真人
有成交意願（要正式報價、下訂、排施工）轉業務，邀客人加公務帳號、建服務群組
設計師／廠商／同業窗口先索取名片、轉業務對接
殺價談折扣、催進度、客訴／抱怨、要發票或簽約付款、大量／專案、情緒激動由真人客服接手處理
AI 不確定的事（有沒有貨、能不能做特殊需求、知識庫沒寫的）不亂答，先說「我幫您確認」並轉真人

📝 詢價管理

管理初步報價階段的案件，尚未安排正式場勘。

狀態流程

詢價需初步估價 → 已初步估價 → 轉場勘 或 標無效

主要操作

按鈕 / 功能說明
＋ 新增案件手動建立一筆新的詢價案件
推進狀態往下推一步（詢價需初步估價 → 已初步估價）
轉為場勘案件案件確定進入場勘流程，移至「場勘管理」
標記無效此案無法繼續，標為無效保存
Pipeline 狀態列點選各階段可篩選該狀態的案件

🔍 場勘管理

管理從安排場勘到報價完成的案件。

狀態流程

待排場勘 → 已排場勘 → 已場勘 → 已建報價資料 → 已發報價單 → 轉成交 或 標無效

主要操作

按鈕 / 功能說明
推進狀態往下推一步
轉為成交案件客戶已同意，移至「成交案件管理」
標記無效此案無法繼續

欄位說明

負責業務 / 客服各角色指派

狀態流程

👥 客戶分類 vs 標籤 — 使用說明

兩者都是整理客戶資料的工具，但用途不同，搭配使用效果最好。

分類（Category）— 客戶的「身份」

每位客戶只屬於一種分類，代表他是什麼類型的客戶。分類會影響系統自動套用的折扣規則與報表統計。

分類名稱適用對象
一般業主個人或家庭委託裝修施工
設計師室內設計師轉介或合作案件
室內裝修公司裝修公司長期合作、批量案件
建設公司建案整批施工，單次金額大
飯店飯店、民宿等商業住宿空間
商業空間辦公室、診所、餐廳等非住宿商業場所
房仲／代銷交屋前翻新需求，由房仲轉介
工廠／廠房工業用途，防焰需求較高

標籤（Tag）— 客戶的「特質或狀態」

一位客戶可以同時擁有多個標籤，用來記錄業務上的觀察與關係狀態，方便快速篩選與 CRM 追蹤。

用途建議標籤說明
消費等級VIP、高消費、一般消費手動升降，配合業績紀錄判斷
來源管道BNI 人脈、Line@ 轉介、客戶介紹、老闆的親友追蹤行銷管道效益
關係狀態回購客戶、潛力客戶、需跟進、冷凍中業務 CRM 管理用
特殊屬性防焰需求、海外客戶、長期合約施工或報價的備注提醒

分類 vs 標籤 — 快速判斷表

問題放分類放標籤
這個客戶是什麼類型？✅—
這個客戶有什麼特質？—✅
一個客戶只能有一個值✅—
一個客戶可以同時符合多個—✅
會影響折扣規則✅—
方便業務快速備注觀察—✅
💡 操作位置：客戶管理頁右上角「管理分類」可新增／編輯分類；「管理標籤」可新增標籤並設定顏色。
