# 專案：繪新內部案件管理系統

> 最後更新：2026/05/21
> 狀態：**開發中 — Phase 1 基礎架構完成**

---

## 專案目標

建立一套內部數位化案件管理系統，讓接單、派工、收款全流程可被追蹤，為連鎖展店提供可複製的營運基礎。

---

## 核心功能需求

### 1. 角色與權限

不同身份有各自的登入介面與可見範圍：
- 業務人員
- 施工師傅
- 排工人員（派單）
- 會計人員
- 管理者（佳樺 / Dan，全覽）

每個人登入後只看到與自己相關的案件與資訊。

---

### 2. 案件資料欄位

每個案件需記錄：

| 欄位 | 說明 |
|------|------|
| 客戶資料 | 姓名、聯絡方式 |
| 需求說明 | 施作項目、範圍、特殊要求 |
| 報價與成交金額 | |
| 收款狀態 | 已付 / 部分付 / 未付 / 待追款 |
| 負責業務 | |
| 施工師傅 | |
| 預定施工日期 | |
| 選用材料 | |
| 案件進度狀態 | 詢價 / 已報價 / 成交 / 排程中 / 施工中 / 完工 / 收款完成 |

---

### 3. 後台管理（管理者視角）

- 所有案件清單，可依狀態、日期、業務、師傅篩選
- 每位人員目前負責的案件一覽

---

### 4. 行事曆派單系統

- 以日曆介面呈現排工狀況
- 每天可看到：哪些師傅已被派工、負責哪些案件
- 未派工案件以獨立清單顯示，方便填補空檔
- **每日業績目標設定**（例如 10 萬 / 15 萬，可自訂）
  - 當日已排業績超標 → 標示顏色（過載警示）
  - 當日已排業績不足 → 標示另一種顏色（偏空警示）
  - 讓排工人員一眼看出哪幾天需要補單或調整

---

### 5. 收付款介面（會計視角）

- 所有案件的收款狀態總覽
- 已收款 / 部分收款 / 未收款 / 逾期追款 分類顯示
- 可標記追款進度與備註
